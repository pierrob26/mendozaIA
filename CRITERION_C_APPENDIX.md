### A1.1 Complex Auction Time Algorithm Implementation

**File**: `AuctionItem.java` (Lines 180-210)
```java
/**
 * Determines if minimum auction time has elapsed based on auction type
 * Demonstrates algorithmic thinking through:
 * - Conditional logic for different business rules
 * - Time calculation algorithms
 * - Edge case handling (null checks)
 */
public boolean hasMinimumTimeElapsed(String auctionType) {
    // Step 1: Input validation (defensive programming)
    if (firstBidTime == null) return false;
    
    // Step 2: Calculate time elapsed using Java Time API
    long hoursElapsed = Duration.between(firstBidTime, LocalDateTime.now()).toHours();
    
    // Step 3: Apply business logic based on auction type
    if ("IN_SEASON".equals(auctionType)) {
        return hoursElapsed >= 24;  // In-season: 24-hour minimum
    } else {
        return hoursElapsed >= 72;  // Off-season: 72-hour minimum
    }
}

/**
 * Composite algorithm that combines time validation with bid status
 * Shows algorithmic composition and logical AND operations
 */
public boolean canBeRemoved(String auctionType) {
    // Combines two boolean conditions using logical AND
    // Player must have bids AND minimum time must have elapsed
    return firstBidTime != null && hasMinimumTimeElapsed(auctionType);
}
```

**Algorithmic Analysis**:
- **Time Complexity**: O(1) - constant time operations
- **Space Complexity**: O(1) - no additional storage required
- **Algorithm Type**: Conditional decision tree with temporal calculations

```java
 * Calculates minimum next bid using graduated pricing algorithm
 * Demonstrates mathematical thinking and business rule implementation
 */
public Double getMinimumNextBid(String auctionType) {
    // Base case: no current bid exists
    if (currentBid == null) {
        // Tiered starting prices based on player classification
        if (getIsMinorLeaguer()) {
            return 0.1;  // $100,000 minimum for minor league players
        } else {
            return 0.5;  // $500,000 minimum for major league players
        }
    }
    
    // Recursive case: current bid + calculated increment
    return currentBid + getMinimumBidIncrement(auctionType);
}

 * Fixed increment algorithm for bid increases
 * Simplified from original complex graduated system for consistency
 */

    if (lastBidTime == null) {
```

**Design Rationale**:
- **Simplicity**: Moved from complex graduated increments to fixed increments for user predictability
- **Fairness**: Same increment rules apply to all players after initial bid
- **Business Logic**: Different starting minimums reflect real-world player valuations

### A2.2 Entity Relationship Modeling

**File**: `UserAccount.java` - Entity Design

```java
@Entity
@Table(name = "users", uniqueConstraints = {
    @UniqueConstraint(columnNames = {"username"})  // Ensures username uniqueness
})
public class UserAccount {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;  // Primary key with auto-increment
    
    @Column(nullable = false, unique = true)
    private String username;  // Business key with uniqueness constraint
    
    @Column(nullable = false)
    private String password;  // BCrypt hashed, never stored in plain text
    
    // Financial tracking fields
    @Column
    private Double salaryCap;         // Team salary limit ($125M default)
    
    @Column
    private Double currentSalaryUsed; // Sum of all player salaries
    
    // Roster management fields
    @Column
    private Integer majorLeagueRosterCount;  // Max 40 players
    
    @Column
    private Integer minorLeagueRosterCount;  // Max 25 players
    
    /**
     * Business logic method - calculates available cap space
     * Encapsulates calculation within the entity
     */
    public Double getAvailableCapSpace() {
        return getSalaryCap() - getCurrentSalaryUsed();
    }
    
    /**
     * Validation method - checks if team can afford a player
     * Implements business rule validation at the data layer
     */
    public boolean canAffordPlayer(Double aas) {
        return aas != null && getAvailableCapSpace() >= aas;
    }
}
```

### A3.1 Multi-Layer Security Architecture

**File**: `SecurityConfig.java`
```java
@Configuration
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            .csrf(csrf -> csrf.disable())  // Disabled for API endpoints
            .authorizeHttpRequests((authz) -> authz
                // Public access patterns
                .requestMatchers("/login", "/register", "/css/**", "/js/**")
                    .permitAll()
                
                // Commissioner-only endpoints (administrative functions)
                .requestMatchers("/auction/manage", "/auction/add-player", 
                                "/auction/remove-player/**")
                    .hasRole("COMMISSIONER")
                
                // Authenticated user endpoints (bidding, viewing)
                .requestMatchers("/auction/view", "/auction/place-bid")
                    .authenticated()
                
                // Default: require authentication
                .anyRequest().authenticated()
            )
            .formLogin((form) -> form
                .loginPage("/login")           // Custom login page
                .defaultSuccessUrl("/", true)  // Redirect after successful login
                .permitAll()
            )
            .logout((logout) -> logout.permitAll());
    }
    
    /**
     * Password encoding configuration
     * Uses BCrypt with default strength (10 rounds)
     */
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
    
    /**
     * Custom user details service
     * Integrates Spring Security with application's user management
     */
    @Bean
    public UserDetailsService users() {
        return username -> {
            UserAccount user = userRepo.findByUsername(username)
                .orElseThrow(() -> new UsernameNotFoundException("User not found"));
            
            return User.builder()
                .username(user.getUsername())
                .password(user.getPassword())  // Already BCrypt hashed
                .roles(user.getRole())         // Maps to Spring Security roles
                .build();
        };
    }
}
```

**Security Features Implemented**:
- **Authentication**: Form-based login with session management
- **Authorization**: Role-based access control (RBAC)
- **Password Security**: BCrypt hashing with salt
- **Session Management**: Automatic session timeout and cleanup
- **CSRF Protection**: Disabled for API endpoints, enabled for forms

## A5. Database Schema and Performance Optimization

### A5.1 Optimized Query Strategies

**File**: `AuctionService.java` - Business Logic Layer
```java
@Service
@Transactional
public class AuctionService {
    
    /**
     * Optimized bid placement with atomic operations
     * Demonstrates transaction management and data consistency
     */
    @Transactional
    public BidResult placeBid(Long auctionItemId, Long userId, Double bidAmount) {
        // Single query to fetch item with lock
        AuctionItem item = auctionItemRepository.findById(auctionItemId)
            .orElseThrow(() -> new RuntimeException("Auction item not found"));
        
        // Validate bid in-memory to reduce database load
        if (!isValidBid(item, bidAmount)) {
            return new BidResult(false, "Invalid bid amount", null);
        }
        
        // Create and persist new bid
        Bid newBid = new Bid(auctionItemId, userId, bidAmount, LocalDateTime.now());
        bidRepository.save(newBid);
        
        // Update item with new current bid (atomic operation)
        item.setCurrentBid(bidAmount);
        item.setCurrentBidderId(userId);
        item.setLastBidTime(LocalDateTime.now());
        auctionItemRepository.save(item);
        
        return new BidResult(true, "Bid placed successfully", newBid);
    }
    
    /**
     * Batch processing for expired auctions
     * Demonstrates efficient bulk operations
     */
    @Scheduled(fixedRate = 3600000) // Run every hour
    public void processExpiredAuctions() {
        List<AuctionItem> expiredItems = auctionItemRepository
            .findByStatusAndLastBidTimeBefore("ACTIVE", 
                LocalDateTime.now().minusHours(24));
        
        // Process in batches to avoid memory issues
        int batchSize = 50;
        for (int i = 0; i < expiredItems.size(); i += batchSize) {
            List<AuctionItem> batch = expiredItems.subList(i, 
                Math.min(i + batchSize, expiredItems.size()));
            processBatch(batch);
        }
    }
}
```

**Performance Optimization Strategies**:
- **Transaction Boundaries**: Minimize transaction scope for better concurrency
- **Batch Processing**: Handle large datasets efficiently
- **Lazy Loading**: Load related entities only when needed
- **Query Optimization**: Use appropriate indexes and query patterns
- **Caching**: Leverage Hibernate second-level cache for read-heavy operations
