PROJECT OVERVIEW
================
FantasyIA is a Fantasy Baseball Dynasty League Management System developed as 
an IB Computer Science Internal Assessment project. This document catalogs all 
sources and origins of code used in this application.

PROJECT METADATA
================
• Project Name: FantasyIA (mendozaIA)
• Framework: Spring Boot 3.2.0
• Language: Java 17
• Database: PostgreSQL
• Architecture: Model-View-Controller (MVC)
• Build Tool: Maven
• Containerization: Docker & Docker Compose

MAIN APPLICATION STRUCTURE
==========================
All core business logic and application-specific code written by project author:

1. MAIN APPLICATION CLASS
   • src/main/java/com/fantasyia/FantasyIaApplication.java
   Purpose: Spring Boot application entry point
   Features: @SpringBootApplication, @EnableScheduling

2. ENTITY CLASSES (ORIGINAL BUSINESS LOGIC)
   Package: com.fantasyia.team
   • Player.java - Baseball player entity with contract management
   • ReleasedPlayer.java - Released players awaiting approval
   • PlayerRepository.java - JPA repository interface
   • ReleasedPlayerRepository.java - JPA repository interface

   Package: com.fantasyia.user
   • UserAccount.java - User authentication & team management
   • UserAccountRepository.java - JPA repository interface
   • RegistrationController.java - User registration logic

   Package: com.fantasyia.auction
   • Auction.java - Auction container entity
   • AuctionItem.java - Individual player auction items
   • Bid.java - Bidding records
   • PendingContract.java - Post-auction contract management
   • AuctionRepository.java - JPA repository interface
   • AuctionItemRepository.java - JPA repository interface
   • BidRepository.java - JPA repository interface
   • PendingContractRepository.java - JPA repository interface
   • AuctionService.java - Core auction business logic
   • AuctionController.java - Web controller for auction management
   • AuctionScheduledTasks.java - Automated auction processing

3. CONTROLLER CLASSES (ORIGINAL WEB LOGIC)
   Package: com.fantasyia.controller
   • HomeController.java - Main navigation controller
   • CustomErrorController.java - Error handling
   • RegisterController.java - Registration flow

   Package: com.fantasyia.team
   • TeamController.java - Team management interface

4. CONFIGURATION CLASSES (ORIGINAL SETUP)

   • DataInitializer.java - Database initialization logic

ARCHITECTURAL DECISIONS
======================
• MVC pattern chosen for separation of concerns
• JPA/Hibernate selected for database abstraction
• Thymeleaf chosen for server-side template rendering
• Spring Security integrated for robust authentication
• Docker containerization for deployment consistency

BUSINESS LOGIC COMPLEXITY
=========================
The application implements sophisticated fantasy baseball rules including:
• Dynamic salary cap management ($125M per team)
• Complex auction timing rules (24hr vs 72hr periods)
• Automated bid processing and contract management
• Roster limit enforcement (40 major, 25 minor league)
• Multi-year contract validation and buyout calculations
